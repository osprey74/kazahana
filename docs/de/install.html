Installationsanleitung

kazahana ist derzeit nicht code-signiert, daher kann Ihr Betriebssystem Sicherheitswarnungen anzeigen.
Folgen Sie den untenstehenden Schritten zur Installation.

desktop_windows Windows

1. Installer herunterladen

Laden Sie den neuesten Installer von der GitHub Releases-Seite herunter.

DateinameFormatZielplattform
kazahana_x.x.x_x64-setup.exeNSIS-InstallerWindows 10 / 11 (64-Bit)
kazahana_x.x.x_x64_en-US.msiMSI-InstallerWindows 10 / 11 (64-Bit)

2. SmartScreen-Warnung behandeln

Beim Ausführen des Installers kann Microsoft Defender SmartScreen eine Warnung anzeigen:

Der Computer wurde durch Windows geschützt
Microsoft Defender SmartScreen hat den Start einer unbekannten App verhindert. Die Ausführung dieser App kann ein Risiko für Ihren PC darstellen.

Dies ist eine Standardwarnung für Apps, die nicht code-signiert sind. kazahana ist Open-Source-Software, deren Quellcode öffentlich auf GitHub verfügbar ist.

  1. Klicken Sie auf “Weitere Informationen” im Warndialog
  2. Der App-Name und die Herausgeberinformationen werden angezeigt
  3. Klicken Sie auf die Schaltfläche “Trotzdem ausführen”

3. Installation

Nach dem Passieren von SmartScreen startet der Standard-Installationsassistent.

  1. Bestätigen Sie den Installationsordner (der Standardwert ist in Ordnung)
  2. Klicken Sie auf “Installieren”
  3. Sobald die Installation abgeschlossen ist, klicken Sie auf “Fertigstellen”

4. Starten

Starten Sie kazahana über das Startmenü oder die Desktop-Verknüpfung. Die SmartScreen-Warnung erscheint nach dem ersten Start nicht mehr.

Hinweis: Browser-Download-Warnungen

  • Microsoft Edge: Kann die Meldung “Diese Datei wird nicht häufig heruntergeladen.” anzeigen. Klicken Sie auf das Menü … und wählen Sie “Behalten”.
  • Google Chrome: Kann die Meldung “Diese Datei ist möglicherweise gefährlich.” anzeigen. Klicken Sie auf das Symbol ∧ und wählen Sie “Behalten”.

laptop_mac macOS

1. Installer herunterladen

Laden Sie den neuesten Installer von der GitHub Releases-Seite herunter.

DateinameZielplattform
kazahana_x.x.x_x64.dmgmacOS (Intel)
kazahana_x.x.x_aarch64.dmgmacOS (Apple Silicon: M1 / M2 / M3 / M4)

So prüfen Sie den Chip Ihres Macs: Klicken Sie auf das Apple-Menü in der oberen linken Ecke und wählen Sie “Über diesen Mac”.

2. App installieren

  1. Öffnen Sie die heruntergeladene .dmg-Datei
  2. Ziehen Sie das kazahana-Symbol in den Ordner Programme
  3. Schließen Sie das .dmg-Fenster und werfen Sie das Disk-Image aus

3. Gatekeeper-Warnung behandeln

Beim ersten Start zeigt macOS Gatekeeper eine Warnung an:

“kazahana” kann nicht geöffnet werden, da der Entwickler nicht verifiziert werden kann.

Dies ist eine Standardwarnung für Apps, die nicht code-signiert oder notarisiert sind.

Methode A: Über Systemeinstellungen erlauben (Empfohlen)

  1. Wenn die Warnung erscheint, klicken Sie auf “OK”, um den Dialog zu schließen
  2. Öffnen Sie Systemeinstellungen > “Datenschutz & Sicherheit”
  3. Unten sehen Sie: “kazahana wurde blockiert, da es nicht von einem identifizierten Entwickler stammt”
  4. Klicken Sie auf “Dennoch öffnen”
  5. Authentifizieren Sie sich mit Ihrem Admin-Passwort oder Touch ID
  6. Wenn der Bestätigungsdialog erscheint, klicken Sie auf “Öffnen”

Methode B: Über Rechtsklick öffnen

  1. Öffnen Sie den Ordner Programme im Finder
  2. Klicken Sie mit der rechten Maustaste (oder Control + Klick) auf kazahana
  3. Wählen Sie “Öffnen” aus dem Kontextmenü
  4. Wenn Sie aufgefordert werden, klicken Sie auf “Öffnen”

Bei einigen macOS-Versionen funktioniert Methode B möglicherweise nicht. Versuchen Sie in diesem Fall Methode A.

Methode C: Quarantäne-Attribut über Terminal entfernen (Fortgeschritten)

Öffnen Sie das Terminal und führen Sie aus:

xattr -cr /Applications/kazahana.app

Dies entfernt das Quarantäne-Attribut (com.apple.quarantine), das auf heruntergeladene Dateien angewendet wird, und ermöglicht es Ihnen, die Gatekeeper-Prüfung zu umgehen.

4. Starten

Starten Sie kazahana über Launchpad oder den Ordner Programme. Nach dem ersten Berechtigungsschritt startet es normal.

delete Deinstallation

Windows

Gehen Sie zu Einstellungen > Apps > Installierte Apps, suchen Sie kazahana und klicken Sie auf “Deinstallieren”.

macOS

Ziehen Sie kazahana aus dem Ordner Programme in den Papierkorb.

build Fehlerbehebung

Die App startet nicht

  • Stellen Sie sicher, dass Ihr Betriebssystem auf die neueste Version aktualisiert ist
  • Windows: Windows 10 oder höher ist erforderlich
  • macOS: macOS 12 (Monterey) oder höher ist erforderlich

“kazahana ist beschädigt und kann nicht geöffnet werden” (macOS)

Führen Sie den folgenden Befehl im Terminal aus:

xattr -cr /Applications/kazahana.app

shield Über die Sicherheit

kazahana ist Open-Source-Software (MIT-Lizenz). Der Quellcode ist öffentlich auf GitHub verfügbar und kann von jedem überprüft werden.

Die App ist derzeit nicht code-signiert, weshalb Ihr Betriebssystem Sicherheitswarnungen anzeigt. Das bedeutet nicht, dass die App gefährlich ist — es bedeutet lediglich, dass der Herausgeber nicht elektronisch verifiziert werden kann. Eine Code-Signierung ist für die Zukunft geplant.
