설치 가이드

kazahana는 현재 코드 서명이 되어 있지 않으므로 OS에서 보안 경고가 표시될 수 있습니다.
아래 단계에 따라 설치하세요.

desktop_windows Windows

1. 설치 프로그램 다운로드

GitHub Releases 페이지에서 최신 설치 프로그램을 다운로드합니다.

파일명형식대상
kazahana_x.x.x_x64-setup.exeNSIS 설치 프로그램Windows 10 / 11 (64비트)
kazahana_x.x.x_x64_en-US.msiMSI 설치 프로그램Windows 10 / 11 (64비트)

2. SmartScreen 경고 처리

설치 프로그램을 실행하면 Microsoft Defender SmartScreen에서 경고를 표시할 수 있습니다:

Windows가 PC를 보호했습니다
Microsoft Defender SmartScreen이 인식할 수 없는 앱의 시작을 차단했습니다. 이 앱을 실행하면 PC가 위험에 처할 수 있습니다.

이것은 코드 서명이 되어 있지 않은 앱에 대한 일반적인 경고입니다. kazahana는 GitHub에서 소스 코드가 공개되어 있는 오픈소스 소프트웨어입니다.

  1. 경고 대화 상자에서 “추가 정보”를 클릭합니다
  2. 앱 이름과 게시자 정보가 표시됩니다
  3. “실행” 버튼을 클릭합니다

3. 설치

SmartScreen을 통과하면 일반 설치 마법사가 시작됩니다.

  1. 설치 폴더를 확인합니다 (기본값 그대로 사용해도 됩니다)
  2. “설치”를 클릭합니다
  3. 설치가 완료되면 “마침”을 클릭합니다

4. 실행

시작 메뉴 또는 바탕 화면 바로 가기에서 kazahana를 실행합니다. 처음 실행 이후에는 SmartScreen 경고가 나타나지 않습니다.

참고: 브라우저 다운로드 경고

  • Microsoft Edge: “일반적으로 다운로드되지 않는 파일입니다.”라고 표시될 수 있습니다. … 메뉴를 클릭하고 “유지”를 선택하세요.
  • Google Chrome: “이 파일은 위험할 수 있습니다.”라고 표시될 수 있습니다. ∧ 아이콘을 클릭하고 “유지”를 선택하세요.

laptop_mac macOS

1. 설치 프로그램 다운로드

GitHub Releases 페이지에서 최신 설치 프로그램을 다운로드합니다.

파일명대상
kazahana_x.x.x_x64.dmgmacOS (Intel)
kazahana_x.x.x_aarch64.dmgmacOS (Apple Silicon: M1 / M2 / M3 / M4)

Mac의 칩을 확인하려면: 왼쪽 상단의 Apple 메뉴를 클릭하고 “이 Mac에 관하여”를 선택하세요.

2. 앱 설치

  1. 다운로드한 .dmg 파일을 엽니다
  2. kazahana 아이콘을 응용 프로그램 폴더로 드래그합니다
  3. .dmg 창을 닫고 디스크 이미지를 추출합니다

3. Gatekeeper 경고 처리

처음 실행 시 macOS Gatekeeper가 경고를 표시합니다:

“kazahana”은(는) 개발자를 확인할 수 없기 때문에 열 수 없습니다.

이것은 코드 서명이나 공증이 되어 있지 않은 앱에 대한 일반적인 경고입니다.

방법 A: 시스템 설정에서 허용 (권장)

  1. 경고가 나타나면 “확인”을 클릭하여 대화 상자를 닫습니다
  2. 시스템 설정 > “개인정보 보호 및 보안”을 엽니다
  3. 하단에 다음과 같이 표시됩니다: “확인된 개발자가 아니므로 kazahana 사용이 차단되었습니다”
  4. “그래도 열기”를 클릭합니다
  5. 관리자 암호 또는 Touch ID로 인증합니다
  6. 확인 대화 상자가 나타나면 “열기”를 클릭합니다

방법 B: 우클릭으로 열기

  1. Finder에서 응용 프로그램 폴더를 엽니다
  2. kazahana를 우클릭 (또는 Control + 클릭)합니다
  3. 컨텍스트 메뉴에서 “열기”를 선택합니다
  4. 확인 메시지가 나타나면 “열기”를 클릭합니다

일부 macOS 버전에서는 방법 B가 작동하지 않을 수 있습니다. 그런 경우 방법 A를 시도하세요.

방법 C: 터미널에서 격리 속성 제거 (고급)

터미널을 열고 다음 명령을 실행합니다:

xattr -cr /Applications/kazahana.app

이 명령은 다운로드된 파일에 적용된 격리 속성(com.apple.quarantine)을 제거하여 Gatekeeper 검증을 우회할 수 있게 합니다.

4. 실행

Launchpad 또는 응용 프로그램 폴더에서 kazahana를 실행합니다. 처음 권한 설정 후에는 정상적으로 실행됩니다.

delete 제거

Windows

설정 > 앱 > 설치된 앱으로 이동하여 kazahana를 찾고 “제거”를 클릭합니다.

macOS

응용 프로그램 폴더에서 kazahana를 휴지통으로 드래그합니다.

build 문제 해결

앱이 실행되지 않습니다

  • OS가 최신 버전으로 업데이트되어 있는지 확인하세요
  • Windows: Windows 10 이상이 필요합니다
  • macOS: macOS 12 (Monterey) 이상이 필요합니다

“kazahana이(가) 손상되었기 때문에 열 수 없습니다” (macOS)

터미널에서 다음 명령을 실행하세요:

xattr -cr /Applications/kazahana.app

shield 보안에 대하여

kazahana는 오픈소스 소프트웨어(MIT 라이선스)입니다. 소스 코드는 GitHub에 공개되어 있으며 누구나 검토할 수 있습니다.

이 앱은 현재 코드 서명이 되어 있지 않아 OS에서 보안 경고가 표시됩니다. 이는 앱이 위험하다는 의미가 아니며 — 단지 게시자를 전자적으로 확인할 수 없다는 것을 의미합니다. 코드 서명은 향후 적용할 예정입니다.
