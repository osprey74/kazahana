Guide d'installation

kazahana n'est pas actuellement signé numériquement, votre système d'exploitation peut donc afficher des avertissements de sécurité.
Suivez les étapes ci-dessous pour l'installer.

desktop_windows Windows

1. Télécharger l'installateur

Téléchargez le dernier installateur depuis la page GitHub Releases.

Nom du fichierFormatCible
kazahana_x.x.x_x64-setup.exeInstallateur NSISWindows 10 / 11 (64 bits)
kazahana_x.x.x_x64_en-US.msiInstallateur MSIWindows 10 / 11 (64 bits)

2. Gérer l'avertissement SmartScreen

Lorsque vous exécutez l'installateur, Microsoft Defender SmartScreen peut afficher un avertissement :

Windows a protégé votre ordinateur
Microsoft Defender SmartScreen a empêché le démarrage d'une application non reconnue. L'exécution de cette application peut mettre votre ordinateur en danger.

Il s'agit d'un avertissement standard pour les applications qui ne sont pas signées numériquement. kazahana est un logiciel open source dont le code source est publiquement disponible sur GitHub.

  1. Cliquez sur « Informations complémentaires » dans la boîte de dialogue d'avertissement
  2. Le nom de l'application et les informations sur l'éditeur seront affichés
  3. Cliquez sur le bouton « Exécuter quand même »

3. Installation

Après avoir passé SmartScreen, l'assistant d'installation standard démarrera.

  1. Confirmez le dossier d'installation (la valeur par défaut convient)
  2. Cliquez sur « Installer »
  3. Une fois l'installation terminée, cliquez sur « Terminer »

4. Lancement

Lancez kazahana depuis le menu Démarrer ou le raccourci sur le bureau. L'avertissement SmartScreen n'apparaîtra plus après le premier lancement.

Note : Avertissements de téléchargement du navigateur

  • Microsoft Edge : Peut afficher « Ce fichier n'est pas couramment téléchargé. » Cliquez sur le menu … et sélectionnez « Conserver ».
  • Google Chrome : Peut afficher « Ce fichier peut être dangereux. » Cliquez sur l'icône ∧ et sélectionnez « Conserver ».

laptop_mac macOS

1. Télécharger l'installateur

Téléchargez le dernier installateur depuis la page GitHub Releases.

Nom du fichierCible
kazahana_x.x.x_x64.dmgmacOS (Intel)
kazahana_x.x.x_aarch64.dmgmacOS (Apple Silicon : M1 / M2 / M3 / M4)

Pour vérifier la puce de votre Mac : Cliquez sur le menu Apple dans le coin supérieur gauche et sélectionnez « À propos de ce Mac ».

2. Installer l'application

  1. Ouvrez le fichier .dmg téléchargé
  2. Glissez l'icône kazahana dans le dossier Applications
  3. Fermez la fenêtre .dmg et éjectez l'image disque

3. Gérer l'avertissement Gatekeeper

Au premier lancement, macOS Gatekeeper affichera un avertissement :

« kazahana » ne peut pas être ouvert car le développeur ne peut pas être vérifié.

Il s'agit d'un avertissement standard pour les applications qui ne sont pas signées numériquement ou notariées.

Méthode A : Autoriser via les Réglages Système (Recommandé)

  1. Lorsque l'avertissement apparaît, cliquez sur « OK » pour fermer la boîte de dialogue
  2. Ouvrez Réglages Système > « Confidentialité et sécurité »
  3. En bas, vous verrez : « L'utilisation de kazahana a été bloquée car il ne provient pas d'un développeur identifié »
  4. Cliquez sur « Ouvrir quand même »
  5. Authentifiez-vous avec votre mot de passe administrateur ou Touch ID
  6. Lorsque la boîte de dialogue de confirmation apparaît, cliquez sur « Ouvrir »

Méthode B : Ouvrir via un clic droit

  1. Ouvrez le dossier Applications dans le Finder
  2. Faites un clic droit (ou Control + clic) sur kazahana
  3. Sélectionnez « Ouvrir » dans le menu contextuel
  4. Lorsque vous y êtes invité, cliquez sur « Ouvrir »

Sur certaines versions de macOS, la Méthode B peut ne pas fonctionner. Dans ce cas, essayez la Méthode A.

Méthode C : Supprimer l'attribut de quarantaine via le Terminal (Avancé)

Ouvrez le Terminal et exécutez :

xattr -cr /Applications/kazahana.app

Cela supprime l'attribut de quarantaine (com.apple.quarantine) appliqué aux fichiers téléchargés, vous permettant de contourner la vérification Gatekeeper.

4. Lancement

Lancez kazahana depuis le Launchpad ou le dossier Applications. Après l'étape d'autorisation initiale, il se lancera normalement.

delete Désinstallation

Windows

Allez dans Paramètres > Applications > Applications installées, trouvez kazahana et cliquez sur « Désinstaller ».

macOS

Glissez kazahana du dossier Applications vers la Corbeille.

build Dépannage

L'application ne se lance pas

  • Assurez-vous que votre système d'exploitation est à jour
  • Windows : Windows 10 ou ultérieur est requis
  • macOS : macOS 12 (Monterey) ou ultérieur est requis

« kazahana est endommagé et ne peut pas être ouvert » (macOS)

Exécutez la commande suivante dans le Terminal :

xattr -cr /Applications/kazahana.app

shield À propos de la sécurité

kazahana est un logiciel open source (Licence MIT). Le code source est publiquement disponible sur GitHub et peut être examiné par n'importe qui.

L'application n'est pas actuellement signée numériquement, c'est pourquoi votre système d'exploitation affiche des avertissements de sécurité. Cela ne signifie pas que l'application est dangereuse — cela signifie simplement que l'éditeur ne peut pas être vérifié électroniquement. La signature numérique est prévue pour l'avenir.
