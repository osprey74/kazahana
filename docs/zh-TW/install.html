安裝指南

kazahana 目前尚未進行程式碼簽署，因此您的作業系統可能會顯示安全性警告。
請按照以下步驟進行安裝。

desktop_windows Windows

1. 下載安裝程式

從 GitHub Releases 頁面下載最新的安裝程式。

檔案名稱格式適用平台
kazahana_x.x.x_x64-setup.exeNSIS 安裝程式Windows 10 / 11（64 位元）
kazahana_x.x.x_x64_en-US.msiMSI 安裝程式Windows 10 / 11（64 位元）

2. 處理 SmartScreen 警告

執行安裝程式時，Microsoft Defender SmartScreen 可能會顯示警告：

Windows 已保護您的電腦
Microsoft Defender SmartScreen 已阻止無法辨識的應用程式啟動。執行此應用程式可能會使您的電腦面臨風險。

這是未進行程式碼簽署的應用程式的標準警告。kazahana 是開放原始碼軟體，其原始碼已公開於 GitHub。

  1. 在警告對話框中點擊「其他資訊」
  2. 將會顯示應用程式名稱和發行者資訊
  3. 點擊「仍要執行」按鈕

3. 安裝

通過 SmartScreen 後，標準安裝精靈將會啟動。

  1. 確認安裝資料夾（使用預設值即可）
  2. 點擊「安裝」
  3. 安裝完成後，點擊「完成」

4. 啟動

從開始功能表或桌面捷徑啟動 kazahana。首次啟動後，SmartScreen 警告將不再出現。

注意：瀏覽器下載警告

  • Microsoft Edge：可能會顯示「此檔案不常被下載。」點擊 … 選單並選擇「保留」。
  • Google Chrome：可能會顯示「此檔案可能有危險。」點擊 ∧ 圖示並選擇「保留」。

laptop_mac macOS

1. 下載安裝程式

從 GitHub Releases 頁面下載最新的安裝程式。

檔案名稱適用平台
kazahana_x.x.x_x64.dmgmacOS（Intel）
kazahana_x.x.x_aarch64.dmgmacOS（Apple Silicon：M1 / M2 / M3 / M4）

如何查看 Mac 的晶片：點擊左上角的蘋果選單，選擇「關於這台 Mac」。

2. 安裝應用程式

  1. 開啟下載的 .dmg 檔案
  2. 將 kazahana 圖示拖曳至應用程式資料夾
  3. 關閉 .dmg 視窗並退出磁碟映像

3. 處理 Gatekeeper 警告

首次啟動時，macOS Gatekeeper 會顯示警告：

無法打開「kazahana」，因為無法驗證開發者。

這是未進行程式碼簽署或公證的應用程式的標準警告。

方法 A：透過系統設定允許（建議）

  1. 當警告出現時，點擊「好」關閉對話框
  2. 開啟系統設定 > 「隱私權與安全性」
  3. 在底部，您會看到：「kazahana 因為不是來自已識別的開發者而被阻擋」
  4. 點擊「強制打開」
  5. 使用管理員密碼或 Touch ID 進行驗證
  6. 當確認對話框出現時，點擊「打開」

方法 B：透過右鍵點擊開啟

  1. 在 Finder 中開啟應用程式資料夾
  2. 在 kazahana 上按右鍵（或 Control + 點擊）
  3. 從快捷選單中選擇「打開」
  4. 當提示出現時，點擊「打開」

在某些 macOS 版本中，方法 B 可能無法使用。如果是這樣，請嘗試方法 A。

方法 C：透過終端機移除隔離屬性（進階）

開啟終端機並執行：

xattr -cr /Applications/kazahana.app

這會移除套用於下載檔案的隔離屬性（com.apple.quarantine），讓您可以繞過 Gatekeeper 驗證。

4. 啟動

從 Launchpad 或應用程式資料夾啟動 kazahana。完成初始權限步驟後，它將正常啟動。

delete 解除安裝

Windows

前往設定 > 應用程式 > 已安裝的應用程式，找到 kazahana，然後點擊「解除安裝」。

macOS

將 kazahana 從應用程式資料夾拖曳至垃圾桶。

build 疑難排解

應用程式無法啟動

  • 請確認您的作業系統已更新至最新版本
  • Windows：需要 Windows 10 或更新版本
  • macOS：需要 macOS 12（Monterey）或更新版本

「kazahana 已損壞，無法打開」（macOS）

在終端機中執行以下命令：

xattr -cr /Applications/kazahana.app

shield 關於安全性

kazahana 是開放原始碼軟體（MIT 授權條款）。原始碼已公開於 GitHub，任何人都可以審閱。

該應用程式目前尚未進行程式碼簽署，這就是您的作業系統顯示安全性警告的原因。這並不代表該應用程式有危險 — 而僅僅表示無法以電子方式驗證發行者。程式碼簽署已計劃在未來實施。
